Rental Car Analytics Pipeline · Priceline

During my co-op at Priceline, I designed and implemented a real-time data pipeline that consumed rental car booking events from Apache Kafka and loaded them into Google BigQuery for downstream analytics. The pipeline processed over 1.2 million events per day and served data to dashboards used by 90+ team members.

One of the more interesting challenges was handling schema drift — rental car data comes from many different suppliers, and field formats weren't always consistent. I built validation and normalization layers to catch discrepancies before they reached BigQuery, which reduced data quality errors significantly and improved trust in the reporting layer.

Tech stack:

  • Apache Kafka (event streaming)
  • Google BigQuery (data warehouse)
  • Java / Spring Boot (pipeline service)
  • SQL (transformation and reporting queries)

Airport Counter Data Quality System · Priceline

Rental car counters at airports are mapped to specific locations in Priceline's system. When those mappings go stale or are misconfigured, customers can end up at the wrong counter. I built a data quality system that automatically flagged these discrepancies using a new RawAirportCounter field, comparing incoming booking data against expected mappings across 100+ airport locations.

The system surfaced roughly 242 affected bookings during initial validation and was integrated into existing monitoring workflows so the ops team could act on alerts in real time. This was my first project owning something end-to-end — from writing the detection logic to presenting findings to the team.

Key outcomes:

  • Detected discrepancies across 100+ airport counter locations
  • Surfaced 242+ potentially affected bookings during validation
  • Integrated into existing ops alerting and monitoring pipelines
  • Presented findings to a cross-functional team of 90+ members

Social Equity Action Lab Website · SEAL @ Northeastern

I'm currently building and maintaining the website for the Social Equity Action Lab (SEAL), a research lab at Northeastern focused on technology and social justice. The site is built with Next.js 14, TypeScript, and Tailwind CSS, and uses Zustand for global state management and Framer Motion for animations.

Working on this project has been a great way to deepen my frontend skills — especially around component architecture and state management patterns. It's also just meaningful work. SEAL does important research, and having a polished, accessible web presence helps them reach more people.

Tech stack:

  • Next.js 14 (React framework)
  • TypeScript (type safety)
  • Tailwind CSS (styling)
  • Zustand (state management)
  • Framer Motion (animations)
